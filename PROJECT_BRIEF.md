# PROJECT_BRIEF.md

## Project name
FundFlow

## Project summary
FundFlow is a finance operations dashboard designed to help users monitor funding utilization, identify budget risks early, detect data quality problems, and export management-ready reports.

## Intended users
Primary users:
- Finance Manager
- Administrative Staff

Secondary users:
- Management / Leadership

## Core problem
Users currently struggle with fragmented financial information, limited forecasting visibility, and data errors that reduce trust in reporting. They need a dashboard that makes the most urgent financial issues visible at a glance and supports fast follow-up actions.

## Core value proposition
FundFlow gives users one clear overview of:
- how much budget has been used
- whether a ceiling breach is likely
- how spending is trending
- where data errors require correction
- how to export insights into a report

## Main user tasks
- View current budget usage
- View days until predicted breach
- Spot data validation issues
- Investigate trends
- Filter by relevant business dimensions
- Export a summary report

## Prototype goal
The prototype should communicate:
- a strong dashboard overview
- clear risk communication
- simple investigation flows
- a credible reporting/export interaction

## Success criteria for the prototype
The prototype is successful if a first-time user can quickly:
1. understand overall financial status
2. notice if action is needed
3. locate the data errors section
4. use filters without confusion
5. export a report confidently
