# CLAUDE.md

You are generating a polished UX prototype in Figma Make for a university assignment.

## Primary goal
Create a desktop web dashboard prototype for a financial administration use case called **FundFlow**. The prototype must look professional, realistic, and suitable for usability testing and screenshots in an academic UX report.

## Context
This prototype supports a student assignment for the role **UX Designer**. The deliverable requires:
- user journey support
- interface mock-ups
- UML use case support
- usability testing with 5 users
- evidence supporting interface quality and usability

This means the design must not only look good, but must also clearly support explainable UX decisions.

## Product concept
FundFlow is a dashboard for finance/admin staff in an SME or public-sector-like setting. It helps users:
- monitor budget usage
- identify risk of budget ceiling breach
- detect and review data validation errors
- inspect spending trends
- filter data by insurer, municipality, or contract
- export reports

## Output expectations
Generate a **desktop-first dashboard prototype** with multiple clear UI states, not just one screen.

At minimum include these screens/states:
1. Overview dashboard
2. Filter state or filter panel
3. Data errors detail state
4. Export report modal/state

Optional additional states if useful:
5. Contract detail view
6. Validation page
7. Reports page

## Design priorities
1. Clarity
2. Professional credibility
3. Low cognitive load
4. Strong visual hierarchy
5. Usability for non-technical office users
6. Good screenshot quality for a report

## UX principles to reflect
In the generated design, visibly support:
- visibility of system status
- consistency and standards
- recognition rather than recall
- minimalist design
- clear error visibility and recovery
- obvious action hierarchy

## Constraints
- Desktop interface only
- Clean modern SaaS style
- Do not make it overly futuristic or flashy
- Avoid dense clutter
- Avoid dark mode
- Avoid excessive animations
- Keep interactions realistic and testable

## Content realism
Use realistic but fictional finance/admin content:
- budget ceiling values
- days until predicted breach
- error counts
- trend percentages
- contract names
- municipality or insurer filters
- export options like PDF and Excel

## Important
The result must feel like a believable MVP for an SME workflow and must be easy to explain in a UX report.
Prefer a small number of strong screens over many weak ones.
