# USABILITY_TEST_PLAN.md

## Purpose
This prototype will be used for a quick usability evaluation with 5 users.

## Therefore the design must support these test tasks clearly

### Task 1
Find the current budget utilization percentage.

Expected success path:
User immediately spots the KPI card on the overview screen.

### Task 2
Determine whether the organization is at risk of exceeding the budget ceiling soon.

Expected success path:
User identifies the "Days Until Breach" card and interprets risk level.

### Task 3
Find where data validation issues need attention.

Expected success path:
User clicks or focuses on the Data Errors card and opens the detail view.

### Task 4
Apply a filter to narrow the dashboard view.

Expected success path:
User finds the filter button without hesitation and understands the filter options.

### Task 5
Export a report.

Expected success path:
User clicks Export Report, chooses a format, and confirms.

## Design implications
The interface must make these tasks easy to perform on a first attempt.
Avoid any design that makes users search unnecessarily.

## What testers should be able to say
- I know where to look first
- I understand what is urgent
- I know how to inspect errors
- I can filter without confusion
- I know how to export the report

## Important
Design with usability testing in mind, not only visual appeal.
