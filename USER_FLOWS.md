# USER_FLOWS.md

## Primary flow 1 — Monitor financial health
User: Finance Manager

Steps:
1. Open dashboard
2. Scan top KPI cards
3. Review budget used
4. Review predicted breach timing
5. Review spending trend chart
6. Decide whether action is needed

Expected UI support:
- KPI cards at top
- strong visual hierarchy
- warning color for risk
- clean chart with actual vs forecast vs ceiling

## Primary flow 2 — Investigate a problem
User: Finance Manager or Administrative Staff

Steps:
1. Notice data errors or unusual trend
2. Click into data errors
3. Review affected records
4. Understand severity and likely issue type
5. Decide on correction or follow-up

Expected UI support:
- clickable data errors card
- clear error list/detail panel
- severity labels
- readable table/list layout

## Primary flow 3 — Narrow analysis with filters
User: Finance Manager

Steps:
1. Open filter panel
2. Filter by insurer, municipality, contract, or time period
3. Apply filters
4. Review updated dashboard state

Expected UI support:
- obvious filter button
- simple dropdowns/chips/selectors
- clear apply/reset actions
- visible selected filter state

## Primary flow 4 — Export report
User: Finance Manager or Management support

Steps:
1. Click Export Report
2. Select export format
3. Confirm report scope
4. Export and receive confirmation

Expected UI support:
- export button top-right
- compact modal
- PDF and Excel options
- confirmation state

## Navigation model
Top navigation can include:
- Overview
- Contracts
- Validation
- Reports

Overview is the home screen and must be the strongest screen.
